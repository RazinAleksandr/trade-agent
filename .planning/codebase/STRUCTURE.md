## Key File Locations

**Entry Points:**
- `main.py`: Autonomous trading loop — orchestrates all 6-step cycle, manages lifecycle, handles SIGINT/SIGTERM shutdown

**Configuration:**
- `config.py`: Single source of truth for all parameters; loads from `.env` via `python-dotenv`
- `.env`: Runtime environment variables (secrets, API keys, trading parameters) — git-ignored, never committed

- `trader.py`: Paper/live execution abstraction, `Trader` class, py-clob-client wrapper, order signing

**State & Monitoring:**
- `data_store.py`: SQLite schema (trades, positions, decisions, market_snapshots, strategy_metrics), CRUD methods
- `portfolio.py`: `PortfolioManager` class — position tracking, P&L calc, risk limits, market resolution
- `logger_setup.py`: `get_logger()` factory, `JsonFormatter` for file output, `log_decision()` for structured logs

**Testing:**
- `tests/test_paper_trading.py`: Smoke tests — imports all modules, validates basic functionality

**Documentation:**
- `docs/architecture.md`: System design, data flow, module responsibilities
- `docs/configuration.md`: All env vars, defaults, tuning guide
- `docs/trading-engine.md`: Kelly criterion derivation, edge calculation, paper vs live
- `docs/api-reference.md`: Gamma API endpoints, CLOB order flow, OpenAI integration
- `docs/wallet-setup.md`: Account generation, funding, L2 credentials, token allowances
- `docs/testing.md`: Test running, coverage, adding new tests

## Naming Conventions

**Files:**
- Module files: `snake_case.py` (e.g., `market_discovery.py`, `data_store.py`)
- Test files: `test_<name>.py` (e.g., `test_paper_trading.py`)
- Config files: `.env`, `.env.example`
- Doc files: `kebab-case.md` (e.g., `trading-engine.md`, `api-reference.md`)

**Functions:**
- Public functions: `snake_case()` (e.g., `fetch_active_markets()`, `analyze_market()`, `generate_signals()`)
- Private functions: `_snake_case()` prefix (e.g., `_parse_market()`, `_passes_filters()`, `_evaluate_signal()`)
- Class methods: `snake_case()` (e.g., `execute_signal()`, `update_position_prices()`)

**Variables:**
- Constants: `UPPER_SNAKE_CASE` (e.g., `MIN_EDGE_THRESHOLD`, `KELLY_FRACTION`, `POLYMARKET_HOST`)
- Instance/local vars: `snake_case` (e.g., `market`, `signal`, `remaining_capital`)

**Types/Classes:**
- Dataclasses: `PascalCase` (e.g., `Market`, `MarketAnalysis`, `TradeSignal`, `OrderResult`)
- Main classes: `PascalCase` (e.g., `Trader`, `PortfolioManager`, `DataStore`)

## Where to Add New Code

**New Feature (e.g., add advanced filtering, new signal type):**
- Primary code: Implement in relevant module (`market_discovery.py`, `strategy.py`, etc.)
- Config params: Add to `config.py` with env var loading and default
- Tests: Add test in `tests/test_paper_trading.py` or create new `tests/test_feature.py`
- Docs: Update relevant guide in `docs/` (e.g., `configuration.md`, `architecture.md`)

**New Module/Service (e.g., add risk management layer):**
- Implementation: Create new `<service>.py` in project root (flat structure, no packages)
- Integration: Import in `main.py`, call during appropriate cycle step
- Config: Add any parameters to `config.py`
- Tests: Add module tests in `tests/test_<service>.py`
- Logging: Use `logger_setup.get_logger(__name__)` at module top

**New Data Table (e.g., track trades by sentiment):**
- Schema: Add table creation to `DataStore._create_tables()` in `data_store.py`
- CRUD: Add methods to `DataStore` class (e.g., `record_sentiment()`, `get_sentiment_history()`)
- Migrations: Manually handle existing DB files (no migration framework)
- Usage: Call from other modules via `store.record_<data>()`

**Utilities/Helpers:**
- Shared functions: Add to module where they fit best (e.g., math helpers in `strategy.py`, parsing in `market_discovery.py`)
- No separate utils file: Keep flat structure; co-locate with usage

## Database Schema

**`trading.db` (SQLite):**

- `trades`: All executed trades (paper + live) — audit log
  - Fields: id, timestamp, market_id, condition_id, token_id, question, side, price, size, cost_usdc, order_id, status, is_paper, estimated_prob, edge, reasoning

- `positions`: Open (and closed) market positions
  - Fields: id, market_id, token_id, question, side, avg_price, size, cost_basis, current_price, unrealized_pnl, realized_pnl, status, opened_at, closed_at
  - Key query: `WHERE status = 'open'` for current portfolio

- `decisions`: Analysis → signal → action chain for every market evaluated
  - Fields: id, timestamp, market_id, question, decision_type, market_price, estimated_prob, edge, kelly_size, action, reasoning, metadata

- `market_snapshots`: Price/volume snapshots at each discovery cycle (audit trail)
  - Fields: id, timestamp, market_id, question, yes_price, no_price, volume_24h, liquidity, end_date

- `strategy_metrics`: Periodic performance summaries (every 5 cycles)
  - Fields: id, timestamp, total_trades, win_rate, total_pnl, avg_edge, sharpe_ratio, metadata

**Location:** `trading.db` in project root (created auto by `DataStore()`)

**Access:** SQLite queries in `data_store.py` via `self.conn.execute()` with row_factory=Row for dict-like access

## Configuration Pattern

All configurable via environment variables in `.env`:

```bash
# Secrets
OPENAI_API_KEY=sk-...
PRIVATE_KEY=0x...

# Polymarket endpoints
POLYMARKET_HOST=https://clob.polymarket.com
CHAIN_ID=137

# Trading mode
PAPER_TRADING=true

# Risk parameters
MAX_POSITION_SIZE_USDC=50
MAX_TOTAL_EXPOSURE_USDC=200
MIN_EDGE_THRESHOLD=0.10
KELLY_FRACTION=0.25

# Market discovery filters
MIN_VOLUME_24H=1000
MIN_LIQUIDITY=500
MAX_MARKETS_PER_CYCLE=10

# Loop timing
LOOP_INTERVAL=300
ORDER_CHECK_INTERVAL=60

# Persistence
DB_PATH=trading.db

# Logging
LOG_LEVEL=INFO
LOG_FILE=trading.log

# Analysis
OPENAI_MODEL=gpt-4o
ENABLE_WEB_SEARCH=true
```

**Loading:** `config.py` uses `python-dotenv` to load `.env` at import time. All modules import `config` and reference params directly (e.g., `config.MIN_EDGE_THRESHOLD`).
