    **lib/pricing.py** (per INST-02, D-09, SAFE-02):
    - Import `ClobClient` from `py_clob_client.client`, `get_logger` from `lib.logging_setup`
    - `get_fill_price(token_id: str, trade_side: str, host: str) -> float`:
      - CRITICAL (Pitfall 1): `get_price()` uses book-side semantics:
        - `get_price(token_id, 'BUY')` = best bid (highest buy order on book)
        - `get_price(token_id, 'SELL')` = best ask (lowest sell order on book)
      - For taker fills (paper trades):
        - If `trade_side == "BUY"`: call `reader.get_price(token_id, "SELL")` to get best ask
        - If `trade_side == "SELL"`: call `reader.get_price(token_id, "BUY")` to get best bid
      - Create unauthenticated `ClobClient(host)` for read-only ops
      - Parse price from result dict: `float(result.get("price", 0))`
      - Raise `ValueError` if price <= 0 (D-10: no fake fills)
      - Include inline comments explaining the inverted semantics
    - `get_best_bid(token_id: str, host: str) -> float`:
      - `reader.get_price(token_id, "BUY")` -- returns best bid
      - Raise ValueError if price <= 0
    - `get_best_ask(token_id: str, host: str) -> float`:
      - `reader.get_price(token_id, "SELL")` -- returns best ask
      - Raise ValueError if price <= 0
    - Do NOT use `get_order_book()` (Pitfall 3: stale data). Use `get_price()` only.
